# GreenBasket API

**API интернет-магазина продуктов с поддержкой категорий, товаров и корзины.**

Проект реализован на Django Rest Framework и предоставляет REST API для работы с каталогом и корзиной пользователя.

## Реализованный функционал

### Категории и подкатегории
- Cоздание, редактирование и удаление через админку.
- Обязательные поля:
  - наименование,
  - slug,
  - изображение.
- Подкатегории связаны с родительской категорией.
- API:
  - получение списка категорий с подкатегориями,
  - получение одной категории (с подкатегориями) по id,
  - поддержка пагинации,
  - поиск по названию (вхождение подстроки),
  - сортировка по названию.

### Продукты
- Управление через админку:
  - создание,
  - редактирование,
  - удаление.
- Продукт содержит:
  - наименование,
  - slug,
  - категорию и подкатегорию,
  - цену,
  - изображения (3 размера).
- API:
  - получение списка продуктов с указанием категории и подкатегории,
  - получение детальной информации об одном товаре по id,
  - поддержка пагинации,
  - поиск названию (вхождение подстроки), 
  - сортировка по названию, подкатегории или категории,
  - фильтрация по категориям и подкатегориям через slug.

### Корзина
- Доступна только авторизованным пользователям.
- Автосоздание корзины.
- API:
  - добавление товара,
  - изменение количества,
  - удаление товара,
  - очистка корзины,
  - просмотр корзины:
    - список товаров,
    - общее количество товаров,
    - общая сумма.

### Аутентификация
- Приложение users реализует:
  - регистрацию пользователя с подтверждением email,
  - JWT-аутентификацию,
  - профиль пользователя,
  - сброс пароля через email.
- Используется библиотека **djangorestframework-simplejwt**.

### Основные сущности

**_Cart_** — корзина для товаров

**_CartItem_** — товар в корзине

**_Category_** — категория

**_SubCategory_** — подкатегория

**_Product_** — товар

**_CustomUser_** — модель пользователя с аутентификацией по email (USERNAME_FIELD)

**_City_** — город

**_SlugMixin_** — модель-миксин для создания slug


### Права доступа

- Запросы для получения категорий, подкатегорий и товаров доступны всем пользователям.

- Доступ к корзине только для авторизованных пользователей.

- Доступ возможен только к своей корзине (кроме суперпользователя).

- Активация аккаунта пользователя после email verification.

- Доступ только к своему профилю (кроме суперпользователя).

## Установка и запуск

1. Клонировать репозиторий
```
git clone https://github.com/simbarilion/GreenBasket.git
cd GreenBasket
```
2. Создать .env (на основе .env.example)
```
SECRET_KEY=your_django_secret_key_here
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1
ALLOWED_URLS=http://localhost,https://localhost,http://127.0.0.1,https://127.0.0.1
DB_NAME=green_basket
DB_USER=postgres
DB_PASSWORD=your_database_password_here
DB_HOST=db
DB_PORT=5432
DJANGO_LOG_LEVEL=INFO
EMAIL_HOST=smtp.yandex.ru
EMAIL_PORT=465
EMAIL_USE_TLS=False
EMAIL_USE_SSL=True
EMAIL_HOST_USER=your_email_host_user_here
EMAIL_HOST_PASSWORD=your_email_host_password_here
```
3. Запуск через Docker
```
docker compose up --build
```
4. После запуска приложение будет доступно по адресу:
```
http://localhost:8000
```

## Архитектурные решения

1. Разделение ответственности (apps).
- Проект разделен на независимые Django-приложения:
  - `catalog` — категории, подкатегории и товары,
  - `cart` — логика корзины,
  - `users` — пользователи и аутентификация.

Это упрощает поддержку и масштабирование проекта.

2. Работа с корзиной.
- Корзина автоматически создается при первом обращении пользователя.
- Корзина жестко привязана к пользователю (OneToOne).
- Все операции с корзиной доступны только владельцу.

Такой подход исключает необходимость вручную создавать корзину и предотвращает доступ к чужим данным.

3. Аутентификация.
- Используется JWT-аутентификация (stateless):
  - не требует хранения сессий на сервере,
  - хорошо подходит для SPA и мобильных клиентов,
  - реализована через `djangorestframework-simplejwt`.

4. Категории и подкатегории
- Подкатегории реализованы как отдельная модель, связанная с категорией через ForeignKey. Это упрощает:
  - фильтрацию,
  - расширение (например, добавление вложенности в будущем).

5. Работа с изображениями.
- Используется `django-imagekit`:
  - автоматическая генерация изображений в нескольких размерах,
  - оптимизация загрузки на клиенте,
  - уменьшение нагрузки на фронтенд.

6. Фильтрация, поиск и пагинация.
- Фильтрация реализована через `django-filter`.
- Поиск реализован через SearchFilter (частичное совпадение).
- Пагинация — стандартная DRF PageNumberPagination.

Это обеспечивает удобную работу с большими наборами данных.

7. Документация API.
- Используется `drf-spectacular`:
  - автоматическая генерация OpenAPI схемы,
  - Swagger UI для тестирования API.

8. Конфигурация через переменные окружения.
- Все чувствительные данные вынесены в `.env`:
  - ключи,
  - доступы к БД,
  - email.

Это повышает безопасность и упрощает деплой.

9. Логирование.
- Реализовано структурированное логирование:
  - отдельные файлы для модулей (`catalog`, `cart`, `users`),
  - ротация логов,
  - уровни логирования через переменные окружения.

10. Контейнеризация.
- Используется Docker:
  - изолированная среда,
  - единообразный запуск,
  - упрощение деплоя.
- Сервисы:
  - web — Django + Gunicorn,
  - db — PostgreSQL.

11. Кастомная команда seed_data для заполнения БД тестовыми данными.

12. Прекоммит-хуки (pre-commit) для поддержания качества кода.
